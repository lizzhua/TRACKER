# 專案目標：AVEDA 品牌情報化自動追蹤與通知系統

本專案依循五大核心架構，旨在建立一個每日自動運作的品牌監測流程：

### 1. 搜集 (Collect)
*   **任務**：使用爬蟲(Scraping)或搜尋 API 獲取最新資訊。
*   **目標來源**：網路新聞、各大公開社群平台與論壇（不限於 Dcard/PTT/Instagram/Threads/Reddit/X 等）、官方公開訊息。
*   **搜尋語系**：中英文 (Chinese and English)。
*   **關鍵字**：AVEDA、肯夢、明星商品(如：木梳、蘊活菁華、Invati、Botanical Repair 等)。

### 2. 分析 (Analyze)
*   **任務**：將搜集到的文本交由 LLM (大型語言模型) 進行自然語言處理。
*   **分析核心**：
    1.  **情緒分析**：判斷討論為正評、負評或中立。
    2.  **活動/資訊萃取**：自動辨識新品上市、促銷、快閃店等重點活動。

### 3. 報告 (Report)
*   **任務**：將 LLM 分析結果結構化，並彙整呈現在動態網頁或儀表板上。
*   **內容重點**：情緒比例、討論熱度 Top 產品、近期活動列表。

### 4. 通知 (Notify)
*   **任務**：將重點摘要主動推播給使用者。
*   **推播管道**：iMessage。
*   **推播內容**：每日情報摘要（整體評價、重大負面預警、新活動通知）。

### 5. 定排 (Schedule)
*   **任務**：設定自動化排程（如 cronjob、GitHub Actions）。
*   **執行頻率**：每日 (Daily) 定時觸發 1~4 之完整流程。
